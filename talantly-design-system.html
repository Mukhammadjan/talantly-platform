Telegram Mini App, bot va admin panel uchun yagona vizual manba. Har bir rang kontrast bo'yicha tekshirilgan, har bir o'lcham shkalada, har bir tegish maydoni 44 px dan katta. Qiymatlar komponentda hardcode qilinmaydi — faqat token orqali chaqiriladi.

01

Prinsiplar

To'rtta qoida. Har bir dizayn qarori shulardan biriga tayanadi. Tayanmasa — chiqarib tashlanadi.

Rang — vazifa, bezak emas.
To'q sariq faqat harakat va brend yuzasi. Yashil faqat tekshirilgan holat. Status, progress, "saqlandi" — hammasi neytral. Shuning uchun ekranda yashil ko'rinsa, u faqat bitta narsani bildiradi.
Telegram — uy egasi.
Header, back, close — Telegram'niki. Biz ularni takrorlamaymiz va ostiga chizmaymiz. Kontent var(--tg-top)dan boshlanadi.
Bitta ekran — bitta asosiy harakat.
Har ekranda bitta to'q sariq tugma. Ikkinchisi kerak bo'lsa — oq yoki ghost. Ikkita to'q sariq tugma = qaror qabul qilinmagan.
Rang ikkiga bo'linadi: yuza va siyoh.
Yorqin rang katta yuzada ishlaydi, matnda ishlamaydi. Shuning uchun har bir rangning -ink varianti bor — kontrast 4.5:1 dan o'tadi.

Bootstrap — har ekranda bir marta
// 1) ready() — BIRINCHI render effect'da. Kech chaqirilsa "Mini App not available".
const tg = window.Telegram?.WebApp;
if (!tg) { showQrFallback(); }   // brauzerda: QR + "Telegramda oching", crash EMAS
tg.ready(); tg.expand(); tg.disableVerticalSwipes();

// 2) BALANDLIK — 100vh ISHLATMA. Telegram header/klaviatura hisobga olinmaydi.
function syncViewport() {
  const r = document.documentElement.style;
  r.setProperty('--tg-vh', tg.viewportStableHeight + 'px');
}
tg.onEvent('viewportChanged', syncViewport);
syncViewport();

// 3) Safe area → CSS o'zgaruvchilarga
function syncInsets() {
  const s = tg.safeAreaInset        || {top:0,bottom:0};
  const c = tg.contentSafeAreaInset || {top:0,bottom:0};
  const r = document.documentElement.style;
  r.setProperty('--tg-top',    (s.top + c.top) + 'px');
  r.setProperty('--tg-bottom', (s.bottom + c.bottom) + 'px');
}
tg.onEvent('safeAreaChanged', syncInsets);
tg.onEvent('contentSafeAreaChanged', syncInsets);
syncInsets();

// 4) Tema QULFLANGAN. themeParams ranglarga ta'sir qilmaydi.
tg.setHeaderColor('#F5F5F7');      // standart — handoff §3.1
tg.setBackgroundColor('#F5F5F7');

// 5) Orqaga — Telegram native. O'z strelkamizni chizmaymiz.
tg.BackButton.onClick(() => history.back());
tg.BackButton.show();
CSS tomonda: .app { height: var(--tg-vh) } — 100vh HECH QACHON. Telegram'da 100vh header va klaviaturani hisobga olmaydi: kontent kesiladi yoki scroll buziladi.
Eslatma: --tg-top qurilmaga qarab 56–110 px orasida o'zgaradi (Dynamic Island, fullscreen, Android). Shuning uchun u doim o'zgaruvchi — hech qachon qat'iy 88 px yozilmaydi. --tg-bottom ham xuddi shunday: nav va CTA unga tayanadi.

09

Menu bar — suzuvchi pill

Qora korpus, ichida dumaloq tugmalar, faol tugma kengayib yorliq bilan to'q sariq pill'ga aylanadi. Bir vaqtda faqat bitta yorliq — foydalanuvchi qayerdaligi bir qarashda o'qiladi.

O'lchamlar: korpus 64, padding 8, tugma 48×48, ikonka 26 — hammasi 4/8 gridda va 44 px tegish chegarasidan yuqori. Nofaol fon #26262E, ikonka #8E8E99 — qorong'i fonda 4.63:1, o'tadi.
Joylashuv: bottom: 16px + var(--tg-bottom). Suzadi. Kontent ostidan o'tishi uchun ro'yxatga padding-bottom:96px.
Anima: kengayish 220 ms cubic-bezier(.2,0,0,1), yorliq 140 ms bilan orqadan kiradi. prefers-reduced-motion yoqilsa — bir zumda.
Rollar (handoff §5): Talant → Asosiy · Testlar · Arizalar · Profil. Izlovchi → Nomzodlar · Doskam · Chat · Ko'proq. Mehmonda Doskam/Chat qulflangan (qulf ikonkasi).
4 tadan oshmaydi. 5-tab kerak bo'lsa — u tab emas, u Profil ichidagi qator.
Nofaol tugmada yorliq yo'q — shuning uchun ularga aria-label majburiy. Aks holda screen reader bo'sh tugma o'qiydi.

12

Do / Don't

Ha: to'q sariq — yuza; action-ink — matn.
Yo'q: 13 px link #F26430 rangida (2.78:1).
Ha: placeholder ink-2.
Yo'q: placeholder ink-3 (3.24:1 — o'qilmaydi).
Ha: progress ink-1 — status.
Yo'q: progress to'q sariq (2.71:1 va qoidani buzadi).
Ha: orqaga — tg.BackButton.
Yo'q: ekran tepasida o'z chizgan ←.
Ha: kontent var(--tg-top)dan.
Yo'q: qat'iy padding-top:88px.
Ha: ikonka-tugmaga 44×44 tegish maydoni.
Yo'q: 20×20 ikonka to'g'ridan-to'g'ri bosiladigan element sifatida.
Ha: shkaladan 7 ta o'lcham.
Yo'q: 14.5 / 15.5 / 12.5 kabi "shunchaki to'g'ri ko'ringan" qiymatlar.
Ha: @media (hover:hover) ichida hover.
Yo'q: mobil ekranda hover'ga tayangan holat (sensorda u yopishib qoladi).
Audit — v1 dan v2 ga nima o'zgardi
Kontrast (8 ta real xato): to'q sariq matn 3.16:1 → action-ink 5.45:1 · to'q sariq soft fonda 2.78:1 → 4.78:1 · yashil badge 3.07:1 → verified-ink 4.80:1 · placeholder ink-3 3.24:1 → ink-2 6.70:1 · xato soft fonda 3.95:1 → danger-ink 5.71:1 · progress 2.71:1 → ink-1 15.3:1.
Tegish maydoni: ko'z ikonkasi 20×20 → 44×44 <button> · chip 36 → 40 · kichik tugma 42 → 44 · checkbox qatori → min 44 · nav tugmasi 52 → 48 (gridda, hali ham 44 dan katta).
Shkala: 15.5 / 14.5 / 13.5 / 12.5 / 11.5 / 34 / 30 / 24 px olib tashlandi — faqat 32·26·20·17·15·13·11 qoldi. Nisbiy line-height:1.5 → qat'iy qiymatlar.
Grid: 18 va 14 oraliqlar → 20 va 16 · input/tugma 54 → 56 (54 to'rtga bo'linmaydi) · nav 66/7/52 → 64/8/48 · gutter ikki faylda 16 va 20 edi — endi ikkalasida 20.
O'zaro ta'sir: barcha :hover → @media (hover:hover) ichiga · hamma interaktiv elementga :focus-visible halqa · <div> kartalar → <button> (klaviatura) · yorliqsiz nav tugmalariga aria-label · CTA sticky · sheet va socials pastiga var(--tg-bottom).
Handoff bilan ziddiyat — qaror kerak 🔥
1. Ikonka: fill yoki stroke?
Handoff §4: "Barchasi FILL (solid) — stroke/outline YO'Q", o'lchamlar nav 24 · header 22 · qator 20 · kichik 16.
Sening so'nggi ko'rsatmang: "iconlarimiz huge icon bo'lishi kerak huddi 5-rasm kabi" — 5-rasm esa HugeIcons Stroke Rounded, 1.5px chiziq.
Ikkalasi bo'lolmaydi. Hozir hujjatda stroke turibdi (so'nggi so'zing). Qaysi biri qoladi?
2. Pastki nav: oq bar yoki qora pill?
Handoff §5: oq fon, tepasida 1px chiziq, 4 ta element, har biriga 10px yorliq doim ko'rinadi, padding 10/8/14, ikonka 24 fill.
Sening so'nggi ko'rsatmang: "4-rasmdagi menu bar bizniki ham shunaqa bo'lsin" — qora suzuvchi pill, faqat faol yorliq ko'rinadi.
Ikkalasi bo'lolmaydi. Hozir hujjatda pill turibdi. Qaysi biri qoladi?
Men o'zim hal qilganlar (rozi bo'lmasang ayt)
Orange hero. Handoff §3.1 setHeaderColor('#F5F5F7') deydi. Hero faqat auth/onboarding'da, o'sha ekranda setHeaderColor('#F26430'). Feed va qolgan hamma joyda #F5F5F7 — handoff o'z kuchida.
Onboarding CTA. Handoff §3.3 bo'yicha ular tg.MainButton bo'ladi. Maketdagi sticky orange tugma — faqat ko'rgazma.
Avatar gradienti. "Gradient yo'q" qoidasidan yagona istisno — handoff §7 shunday belgilagan, u identifikator.
Rasmiy tokenlardagi ikkita a'zolik muammosi
Focus halqasi ko'rinmaydi. Handoff §2.6: 0 0 0 3px rgba(242,100,48,.22). 22% shaffoflik oq fonda ≈ #FCE0D5 — kontrast ~1.1:1. Ya'ni klaviatura bilan yurgan odam qayerdaligini ko'rmaydi (WCAG 2.4.11 uchun 3:1 kerak). Men rgba(184,67,26,.34) qo'ydim. Tasdiqlaysanmi?
Oq matn #F26430 ustida = 3.16:1. Brend rangi qulflangani uchun tegmadim, lekin uchta shart bilan: yorliq 17/700, 3:1 UI-komponent chegarasi, va 15px dan kichik matnda hech qachon. Kichik matn kerak bo'lsa — --action-ink #B8431A (5.45:1).
